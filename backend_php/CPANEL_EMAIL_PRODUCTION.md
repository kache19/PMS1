## How It Works

1. **Email Account**: `no-reply@malenyapharmacy.com` 
   - This is a cPanel-hosted email account on your domain
   - All system emails (password resets, notifications, etc.) come from this address
   - Replies go to this mailbox

2. **SMTP Setup for PHP**
   - Host: `mail.malenyapharmacy.com` (Your cPanel mail server)
   - Port: `587` (TLS encryption for security)
   - Encryption: TLS (more reliable than SSL on port 465)
   - Authentication: Full email address + password

3. **Email Delivery Flow**
   ```
   PHP Application
      ↓
   PHPMailer Library
      ↓
   mail.malenyapharmacy.com:587 (TLS)
      ↓
   Recipient's Email Server
      ↓
   Recipient Inbox
   ```

## Troubleshooting

### Issue: "Connection refused" or timeout
**Solution:**
1. Verify `no-reply@malenyapharmacy.com` exists in cPanel
2. Check email account is active (not suspended)
3. Verify password is correct in `.env`
4. Check firewall allows port 587 outbound

**Command to test connection (SSH):**
```bash
telnet mail.malenyapharmacy.com 587
```
Should respond with: `220 mail.malenyapharmacy.com ESMTP`

### Issue: "Authentication failed"
**Solution:**
1. Verify username is full email: `no-reply@malenyapharmacy.com`
2. Check password has no special characters that need escaping
3. If password contains special chars, update cPanel password
4. Verify email account in cPanel hasn't been deleted

### Issue: Emails going to spam/junk
**Solutions:**
1. Add SPF record: `v=spf1 include:mail.malenyapharmacy.com ~all`
2. Add DKIM record (generated by cPanel)
3. Add DMARC record: `v=DMARC1; p=none; rua=mailto:admin@malenyapharmacy.com`
4. Use "Malenya Pharmacy" as sender name (not generic "noreply")

**cPanel Steps:**
1. Go to cPanel → Email Authentication
2. Enable SPF, DKIM, DMARC
3. Wait 24 hours for DNS propagation

### Issue: Email account locked after multiple failed attempts
**Solution:**
1. Wait 1-2 hours for automatic unlock
2. OR reset password in cPanel
3. Update `.env` with new password

## Switching Email Accounts

If you want to use a different cPanel email account:

1. **Create new email in cPanel**
   - cPanel → Email Accounts → Create
   - Example: `notifications@malenyapharmacy.com`

2. **Update `.env`**
   ```ini
   MAIL_USERNAME=notifications@malenyapharmacy.com
   MAIL_PASSWORD=account_password
   MAIL_FROM_ADDRESS=notifications@malenyapharmacy.com
   MAIL_FROM_NAME=Malenya Pharmacy Notifications
   ```

3. **Test immediately**
   - Visit: `test_email_web.php?email=your-test@gmail.com`
